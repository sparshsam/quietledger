## Overview

Ten areas of platform hardening, all local-first, all backward-compatible with existing v3 schema data.

## Implementation Plan

### 1. IndexedDB Storage Engine
**Files:** `src/lib/data/storage.ts` (new), `src/lib/data/types.ts`, `src/lib/data/persistence.ts`

Replace localStorage single-JSON-blob with IndexedDB for the primary ledger store:
- Asynchronous get/set/delete operations
- Transactional updates — partial saves, no full-rewrite
- 5MB+ localStorage limit removed
- Graceful fallback to localStorage when IndexedDB unavailable
- Migration: on first load, read from localStorage, write to IndexedDB, clear localStorage key
- Same schema version, same data shape — wire-compatible

### 2. Offline Detection & Indicator
**Files:** `src/lib/hooks/useOnlineStatus.ts` (new), `src/app/globals.css`, `src/app/app/page.tsx`

- Custom hook: `useOnlineStatus()` — tracks `navigator.onLine` + `online`/`offline` events
- Offline banner: fixed-position bar at top when offline, auto-dismisses on reconnect
- SW: Add message handler for `SKIP_WAITING`, improve cache strategies

### 3. PWA Install Polish
**Files:** `src/components/pwa-install.tsx` (new), `src/app/app/page.tsx`, `src/app/globals.css`

- Listen for `beforeinstallprompt` event
- Show install button in navbar/app when installable
- Handle iOS standalone detection, show install instructions for Safari
- Track if already installed (standalone mode check)

### 4. Update Flow Improvements
**Files:** `src/components/pwa-register.tsx`, `public/sw.js`

- Add message event listener in SW for `SKIP_WAITING`
- Improve update banner: show version number, changelog link
- Add "Dismiss" option to postpone update
- Reduce poll interval for production

### 5. Performance — Code Splitting
**Files:** `src/app/app/page.tsx`

- Lazy-load tab components with `next/dynamic`:
  - `LedgerReport`
  - `TransactionsView` + `ImportFlow`
  - `AccountsView`
  - `GoalsPanel` + `BudgetsPanel` + `RecurringPanel`
  - `ReportsView`
  - `SearchView`
  - `QuickJump`
- Each tab loaded on first visit, cached after

### 6. Accessibility Improvements
**Files:** `src/app/globals.css`, `src/app/app/page.tsx`

- Add `aria-live="polite"` region for dynamic updates (save notices, errors)
- Ensure keyboard focus is managed when tabs switch
- Add dark mode contrast improvements for small text
- Ensure all interactive elements have visible focus indicators
- Add reduced motion support for animations

### 7. Error Recovery & Crash Resilience
**Files:** `src/components/error-boundary.tsx`, `src/lib/data/persistence.ts`, `src/app/app/page.tsx`

- Enhanced error boundary: add "Reset section" button, auto-retry with backoff
- Auto-save: incrementally save to temp key on every mutation (crash recovery)
- On load: detect stale temp data, offer recovery
- Add storage integrity check: validate saved data integrity on load

### 8. Security Review
**Files:** `next.config.ts`, `src/middleware.ts`, `public/sw.js`

- CSP: tighten `script-src` with nonce/hash for inline scripts where possible
- Add `Cross-Origin-Resource-Policy: same-origin`
- Add `Cross-Origin-Opener-Policy: same-origin`
- Review and document all third-party requests

### 9. Documentation Refresh
**Files:** `ARCHITECTURE.md`, `CHANGELOG.md`, `CLAUDE.md`, `docs/architecture.md`

- Update ARCHITECTURE.md with v0.11 storage layer
