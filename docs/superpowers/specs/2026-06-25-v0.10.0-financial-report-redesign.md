# OpenLedger v0.10.0 — Financial Report Redesign

> Turning the Ledger tab into a connected financial report. Import → Categorize → Analyze → Budget → Track.
>
> Design system: OpenProof Design Playbook (desktop canonical).  
> Brand: warm parchment + deep ledger brown `#7A2F00`.  
> Metaphor: financial journal — you record, it reveals.

## Overview: 4 Connected Phases

```
Import CSV → Smart Categorize → Preview & Accept
  → Ledger tab now shows your month (income, expenses, trends, categories)
    → Tap a category → filtered transactions appear below
      → "Based on your $682 grocery spend, set a $600 budget?"
        → Budgets pre-populated with real numbers
          → Next month: budget vs actual on the Ledger tab
```

Every phase connects to the next. None are separate pages.

## Phase 1 — Import Flow Redesign

### Current Problem
Import saves directly with no review step. Categories are unassigned. User has no control over what enters their ledger.

### Design
A vertical staged flow (playbook §4 — Action Pages):

```
[Import transactions] pill on Ledger tab

===========================
| Step 1: Select account  |
| [Select ▼] [+ New]      |
|                          |
| Step 2: Upload file      |
| [rounded-2xl drop zone]  |
| no dashed border         |
|                          |
| Step 3: Review & Accept  |
| ┌──────────────────────┐ │
| │ Date  │ Desc   │ Cat │ │
| │──────────────────────│ │
| │ Jun 24│ Metro  │ Groc│ │  ← dropdown per row
| │ Jun 23│ Uber   │ Food│ │
| └──────────────────────┘ │
|                          |
| [Import X transactions]  |
===========================|

Save behavior:
- Accepted rows → saved to localStorage
- Transactions marked `source: "csv"` + import ID metadata
- On save → redirect to Ledger tab showing updated month
- Sticky banner: "X transactions imported — your month has changed"
```

### Auto-Categorization
The existing `categoryKeywords` map in `csv-import.ts` is applied during preview:

| Keyword pattern | Category |
|----------------|----------|
| loblaws, metro, costco, supermarket | Groceries |
| uber eats, doordash, skip, takeout | Food delivery |
| uber, lyft, transit, presto, gas, fuel | Transport |
| netflix, spotify, apple.com/bill, patreon | Subscriptions |
| salary, payroll, deposit, paycheque | Income |
| hydro, internet, mobile, phone, insurance | Utilities |
| amazon, walmart, ikea | Shopping |
| pharmacy, clinic, doctor, dental | Health |

Each preview row shows the detected category as a `<select>` the user can override. New patterns can be added to the map at any time.

### Key Math: The Summary Strip

The hero is no longer net worth — it's the **month's financial delta**:

| Metric | Computation |
|--------|------------|
| **Income** | Sum of all `amount > 0` where `date` starts with selected month |
| **Spent** | Absolute sum of all `amount < 0` where `date` starts with selected month |
| **Remaining** | `Income - Spent` |
| **vs Last Month** | Compare current month's total to previous month's total |

This is computed from the `totals.ts` engine (already built — `computeIncome`, `computeExpenses`, `computeNetCashflow`). The month selector controls the filter.

### Chart Rules

- Charts are inline SVG (existing components in `src/components/charts/`)
- Max 2 charts visible at once on mobile, 4 on desktop
- Charts are compact — no legends, no gridlines, no axis labels (data labels only)
- Chart colors use the brand palette — `#7A2F00` for positive, a parchment-warm red for negative
- Empty chart state: single line of text ("No data this month")
- Selecting a chart element filters the category breakdown and transaction list below
- The Spending by Category chart uses a donut variant for visual variety

### Interactive Filtering

- Tapping a category row in the breakdown filters the transaction table below to that category
- Tapping the same category again removes the filter
- The month selector updates ALL data — summary, charts, breakdown, transactions
- Filter state is shown as an inline pill ("Filtered: Groceries [×]") — not a box or panel

### Empty State (first use)

```
            Your ledger is empty
    Import your first bank statement to begin
              [Import transactions]
```

No icon. No dashed border. No illustration. Text on canvas + one pill. (Playbook §4)

## Phase 3 — Budgets From Import Data

### Current Problem
Budgets are created from blank forms with no connection to historical spending.

### Design — Budget Suggestion Engine

When a user visits the Goals tab (or the budget section on Ledger), if they have ≥1 month of import data, show:

```
Based on your average spending:

Groceries     $620/mo  [Set budget →]
Transport     $340/mo  [Set budget →]
Food delivery $250/mo  [Set budget →]
```

Each [Set budget →] pill opens the budget creation form pre-filled with:
- Category: the detected category
- Amount: the 3-month average (or 1-month if only 1 month of data)
- Month: current month

### Computation

```typescript
function averageSpendingByCategory(transactions: Transaction[], months: number): CategoryAvg[] {
  // Group transactions by category for the last N months
  // Filter to expenses only (amount < 0)
  // Average the monthly totals per category
  // Return sorted by highest average first
}
```

This reuses the existing `categoryTotals` and `monthlyTotals` from the finance engine.

### Budget Progress on Ledger Tab

When budgets exist, they appear as a compact list below the category breakdown:

```
Budget           Planned   Spent    Left
Groceries        $600      $682     -$82  ▓▓▓▓▓▓▓▓
Transport        $400      $312     $88   ▓▓▓░░░░
```

Progress bars use the `budgetUtilization` / `remainingBudget` / `isOverBudget` helpers (already built in `budgets.ts`).

## Phase 4 — Connection & Consistency

### Math Must Be Tight

Every number on every page must derive from the same source state and use the same computation path:

| Component | Current file | What it computes |
|-----------|-------------|-----------------|
| `totals.ts` | `src/lib/finance/totals.ts` | Income, expenses, cashflow, net worth |
| `grouping.ts` | `src/lib/finance/grouping.ts` | Category totals, monthly totals |
| `insights.ts` | `src/lib/finance/insights.ts` | Largest expense, top category, month-over-month |
| `trends.ts` | `src/lib/finance/trends.ts` | Trend data for charts |
| `budgets.ts` | `src/lib/finance/budgets.ts` | Utilization, remaining, over-budget detection |
| `goals.ts` | `src/lib/finance/goals.ts` | Goal progress percentage |

**Pre-audit check:** All Ledger tab numbers must pass through these same helper functions. No inline `.filter().reduce()` that duplicates logic. If a new computation is needed, add it to the finance engine (with a test), not inline in the component.

### Data Flow

```
localStorage  ──→  loadLedgerState()  ──→  React state
                                              │
                                    Finance engine helpers
                                    (totals, grouping, insights, budgets)
                                              │
                                              ▼
                                    Ledger tab renders:
                                    - Summary strip (from totals)
                                    - Charts (from trends + grouping)
                                    - Category breakdown (from grouping)
                                    - Transactions (filtered by state)
                                              │
                                    User action (import, filter, select)
                                              │
                                              ▼
                                    saveLedgerState() → localStorage
```

- Cloud backup & restore

## Visual Style Rules

Following the design playbook strictly:

**No card containers.** Content sits on the parchment canvas (`#F5F0E8`). Sections separated by generous whitespace (playbook §3 — spacing rhythm).

**Summary strip:** `border-t border-b border-[rgba(0,0,0,0.06)] py-4` — three values inline with large bold numbers. Label is `text-xs font-bold tracking-wider uppercase text-text-muted`.

**Category breakdown:** Grid layout. Each row: `flex justify-between items-center py-3 border-b border-[rgba(0,0,0,0.06)]`. Category name, amount, vs-last-month change (colored), optional mini progress bar.

**Charts:** No legends. No gridlines. Data labels only (the number). Compact — `max-h-[180px]` on mobile, `max-h-[240px]` on desktop.

**Interactive filter:** When a category is selected, an inline pill shows at the top of the transactions section: `"Filtered: Groceries [×]"`. No panel, no drawer.

**Buttons:** All pills. Primary = `bg-[#7A2F00] text-white`. Secondary = `bg-[#EBE5D8] text-[#3A2A1A]`. No glow, no scale.

**Empty states:** Text only (playbook §3). "Import your first bank statement" for empty ledger.
