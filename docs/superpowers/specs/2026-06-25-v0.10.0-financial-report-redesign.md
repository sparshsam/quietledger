# OpenLedger v0.10.0 — Financial Report Redesign

> Turning the app into a connected financial report. Import → Categorize → Understand → Budget → Goals.
>
> Design system: OpenProof Design Playbook (desktop canonical).  
> Brand: warm parchment + deep ledger brown `#7A2F00`.  
> Metaphor: a financial journal — you record, it reveals.

---

## App Identity Card (Playbook §9)

| Property | Definition |
|----------|-----------|
| **Core metaphor** | A financial journal — you record, it reveals |
| **Primary brand color** | `#7A2F00` (deep ledger brown) |
| **Icon usage** | Wordmark in nav only |
| **Typography personality** | Stack Sans Notch (warm editorial), crisp mono for numbers. **Numbers dominate — amounts are always larger than their labels.** |
| **Emotional tone** | Calm, editorial, trustworthy |
| **Main user action** | Understanding where your money went |

---

## Overview: Connected Workflow

```
Import → Categorize (auto, learns from corrections) → Understand ("Where did my money go?")
  → Set budgets from real averages → Track vs actual → Set goals after understanding
```

Every phase connects to the next. The app is a pipeline, not a collection of pages.

---

## Navigation (5 Tabs)

| Tab | Purpose |
|-----|---------|
| **Ledger** | The monthly financial report. Newspaper-like. Charts, categories, transactions. |
| **Transactions** | Full transaction list with search, filters, CRUD. |
| **Accounts** | Account management hub. Account list with balances. Tap filters the entire app. |
| **Goals** | Budgets + goals. Budget creation with avg-spending suggestions. Goals come after. |
| **Settings** | Profile, data management, MCP tokens, privacy, legal. |

**Behaviour:** All tabs share a common filter context — the selected month, account, and category filter propagate everywhere. Selecting an account in Accounts updates Ledger, Transactions, and Goals.

## Phase 1 — Import as Onboarding

### Current Problem
Import saves directly with no review. Feels like a utility, not part of the product.

### Philosophy
Import should feel like **"bringing your financial life into OpenLedger."** A guided experience, not a spreadsheet upload.

### Flow

```
[Import transactions] pill on Ledger tab
```

A vertical staged sheet (playbook §4 — Action Pages):

```
┌────────────────────────────────────────┐
│  Bring your financial life             │
│  into OpenLedger                       │
│                                        │
│  Step 1 — Choose Account               │
│  [Select account ▼]  or  [+ New]       │
│  (Chequing, Savings, Credit...)        │
│                                        │
│  Step 2 — Upload Statement             │
│  [rounded-2xl drop zone — no border]   │
│  "Bank export, CSV or TSV"             │
│                                        │
│  Step 3 — Preview & Accept             │
│  Auto-categorized from description     │
│  ┌──────────────────────────────────┐  │
│  │ Jun 24  Metro Grocers  Groceries │  │  ← editable category
│  │ Jun 23  Uber Eats      Food      │  │  ← dropdown per row
│  │ Jun 22  Starbucks      Coffee    │  │
│  │ Jun 21  Shell Gas      Fuel      │  │
│  └──────────────────────────────────┘  │
│                                        │
│  Import 47 transactions →              │
└────────────────────────────────────────┘
```

**Save behaviour:**
- Accepted rows → saved to localStorage with `source: "csv"` + import ID
- On save → Ledger tab opens showing the updated month report
- A single sentence appears: "47 transactions imported — your month has changed"
- The "Where did my money go?" section now has data

### Category Learning (First-Class Feature)

Every correction the user makes teaches the system:

```
User imports: STARBUCKS → detected "Food|Restaurants"
User changes to: "Food|Coffee"
→ System remembers: STARBUCKS → "Food|Coffee"
→ Next import of STARBUCKS → already "Food|Coffee"
```

**Persistence:**
- `localStorage` key: `openledger_category_learnings: Array<{ pattern: string; category: string; subcategory: string }>`
- For signed-in users: synced to `openledger_sync_events` for cross-device learning
- User can view/edit learned mappings in Settings → Data

### Key Principles

**Numbers as typography.** The amount is always larger and appears before the label:
- `$4,820` in `text-3xl font-black` then `Income` in `text-xs uppercase text-secondary` below it
- Never "Income $4,820" — always "$4,820" then "Income"

**Transactions occupy at least half the screen** after summaries. Charts summarize, transactions explain.

**Month comparison on every metric.** Every number shows `vs last month` delta:
- Positive (expense went down): green/muted with ▼
- Negative (expense went up): warm amber with ▲
- Income going up: ▲ is good. Income going down: ▼ is neutral/warning.

### The "Where Did My Money Go?" Section (Signature Feature)

This is the first thing the user sees after the summary strip. It's a simple category breakdown:

```
Where did my money go?

Food               $642  ▲18%
Housing           $1,100   —
Transport          $320  ▼6%
Subscriptions       $84  ▲2%
Shopping           $518  ▲41%
```

One glance. That's the app. The percentages are vs last month.

Tapping any row filters everything below (charts, transactions) to that top-level category.

### Interactive Filtering

Three filter axes, all shared across tabs:

| Axis | Source | Behaviour |
|------|--------|-----------|
| **Month** | Pill selector at top | Updates ALL data on every tab |
| **Account** | Pill selector + Accounts tab | Filters to one account's data |
| **Category** | Tap category breakdown row | Filters transactions + charts |

Filter state is shown as an inline pill: `"Filtered: Food [×]"` — not a box or panel. Multiple filters combine (AND).

Tapping a chart element (donut segment, trend point) also sets the category filter. The app feels alive.

## Phase 4 — Budgets & Goals (in Order)

### Correct Ordering

Goals (savings targets) only make sense AFTER budgets (spending limits). The Goals tab shows:

1. **Budgets** — from spending averages. "Based on your data, you typically spend..."
2. **Goals** — savings milestones. Created after budgets are set.

### Budget Suggestions From Import Data

When the user has ≥1 month of import data, the Goals tab shows:

```
Based on your average spending:

Groceries     $680/mo  [Set budget]
Transport     $320/mo  [Set budget]
Food          $420/mo  [Set budget]
Shopping      $310/mo  [Set budget]
```

Each [Set budget] opens the form pre-filled with category and amount (3-month average, or 1-month fallback).

### Budget Progress on Ledger Tab

Compact inline list:

```
Groceries     ▓▓▓▓▓▓   75%    $600 budget
Transport     ▓▓▓░░░   80%    $400 budget
```

### Goal Suggestions

Goals appear as suggestions after budgets are set:

```
Budgets are set. Ready for your next step?

Emergency Fund     $0 / $10,000  [Start goal]
New Laptop         $0 / $2,500   [Start goal]
```

## Finance Engine Rules (§14 — Immutable)

**Every financial value displayed anywhere in the application must come from the finance engine.**

No component may compute:
- Totals (income, expenses, cashflow, net worth)
- Balances (account + transaction totals)
- Averages (spending by category, month-over-month)
- Budget metrics (utilization, remaining, over-budget)
- Category totals
- Trends (monthly, daily, weekly)
- Comparison deltas (vs last month, vs budget)

**All computation goes through helper functions in `src/lib/finance/`.**

| Helper | File | Computes |
|--------|------|----------|
| `computeIncome(transactions, month?)` | `totals.ts` | Income for all time or a specific month |
| `computeExpenses(transactions, month?)` | `totals.ts` | Expenses for all time or a specific month |
| `computeNetCashflow(transactions, month?)` | `totals.ts` | Income - Expenses |
| `computeNetWorth(accounts, transactions)` | `totals.ts` | Net worth |
| `computeMonthOverMonth(transactions, category?)` | `totals.ts` | Current vs previous month delta for all or one category |
| `categoryTotals(transactions, month?)` | `grouping.ts` | Spending per category |
| `monthlyTotals(transactions)` | `grouping.ts` | Income/expense per month |
| `largestExpenseThisMonth(transactions)` | `insights.ts` | Single largest expense |
| `topSpendingCategory(transactions)` | `insights.ts` | Category with highest total |
| `monthOverMonthChange(transactions)` | `insights.ts` | Total expense change vs last month |
| `findRecurringTransactions(transactions)` | `insights.ts` | Transactions appearing 2+ times |
| `budgetUtilization(budget, transactions)` | `budgets.ts` | % of budget spent |
| `remainingBudget(budget, transactions)` | `budgets.ts` | Budget - spent |
| `isOverBudget(budget, transactions)` | `budgets.ts` | Boolean |
| `findOverBudget(budgets, transactions)` | `budgets.ts` | List of over-budget items |
| `averageSpendingByCategory(transactions, months)` | `budgets.ts` | Monthly average per category (NEW) |
| `goalProgress(goal)` | `goals.ts` | % of target reached |

**Adding a new metric:** Write the helper in the finance engine. Write a test. Import the helper in the component. Never inline.

## Chart Rules

- Inline SVG (existing components in `src/components/charts/`)
- Max 2 charts visible at once on mobile, 4 on desktop
- No legends, no gridlines, no axis labels — data labels only
- Colors use brand palette: `#7A2F00` for positive, warm parchment-red for negative
- Empty chart state: single text line ("No data this month")
- **Contextual:** selecting a chart element (donut segment, trend point) sets the category filter. All other charts + transactions update.
- Charts are compact: `max-h-[200px]` mobile, `max-h-[300px]` desktop

---

## Visual Style Rules (Playbook-Compliant)

- **No card containers** — content sits on parchment `#F5F0E8`. Sections separated by whitespace.
- **Numbers dominate** — amounts are `text-3xl` to `text-5xl font-black`. Labels are `text-xs font-bold tracking-wider uppercase text-text-muted` below the number.
- **Summary strip:** `border-t border-b border-[rgba(0,0,0,0.06)] py-4`.
- **Category breakdown rows:** `flex justify-between items-center py-3 border-b border-[rgba(0,0,0,0.06)]`. Name, amount (bold), vs-last-month delta (colored). Tap to filter.
- **Buttons:** All pills. Primary = `bg-[#7A2F00] text-white`. Secondary = `bg-[#EBE5D8] text-[#3A2A1A]`.
- **Empty states:** Text only (playbook §3). "Import your first bank statement to begin."
- **Negative space is our best friend.** If something isn't the main thing, it shouldn't be on screen.
